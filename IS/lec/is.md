## Генераторы ПСП (псевдослучайных последовательностей) на основе остаточных вычислений (по модулю)
на основе линейного конгруэнтного генератора, описыающегося рекуррентным соотношением: xt+1=(a*xt+c)midN, x0 - начальное значение ПСП,a - множитель, c - приращение, N - можщность алфавита  
**Генератор на основе алгоритма BBS** (с квадратичным остатком)  
1. Выбираются 2 простых числа p, q (как в RSA) (должны быть сравнимы 3 по модулю 4, pmod4=3, qmod4=3)
2. Вычисляется число n=p*q (целое число Блюма)
3. Выбирается другое случайное целое число x, взаимно простое с n
4. Вычисляется число x0=(x^2)mod n (стартовое число генератора)
5. на каждом шаге вычисляется xi+1=(xi^2)mod n
6. Результатом n-го шага обычно является крайний (младший) бит xi

# Введение в асимметричную криптографию
У симметричной криптографии 2 проблемы: согласования (как передать ключ так, чтобы его никто не перехватил), нужен новый ключ для каждого собеседника  
**Диффи, Хеллман в 1976 предложили алгоримт согласования информации между двумя абонентами, и это положило начало асимметричной криптографии (криптографии с открытым ключом)**  
Преимущество: только один ключ у каждого абонента, группы и т.д. Ключ делится на 2 части: публичный и приватный  

Первообразный корень (primary root): A является первообразным корнем по модулю P в уравнении a^x mod p = y, если, изменяя x от 1 до p, мы получаем неповторяющиеся остатки [1; p-1]. Например, a=2, p=5: 2^1%5=2; 2^2%5=4; 2^3%5=3; 2^4%5=1 - следовательно, 2 - первообразный корень по модулю 5.  
Криптостойкость алгоритма Диффи-Хеллмана связана с трудностью решения задачи дискретного логарифмирования в конечном поле  
Алгоритм не защищён от атак man-in-the-middle, если стороны не предпринимают дополнительных мер  

Асимметричный алгоритм на основе задачи об укладке ранца (ранцевый алгоритм)  
Сверхвозрастающея является последовательность, где последующий член больше сумме всех предыдущих  
Открытый текст/шифруемое сообщение ассоциируется с предметами Mi из утсановленного множества  
Существуют 2 подхода к укладке ранца: простой (соотносится с тайным ключом) и сложный (соотносится с открытым ключом)  
В асимметричных системах отправитель всегда шифрует отправляемое сообщение публичным ключом получателя. Получатель расшифровывает полученное сообщение своим тайным ключом  
В качестве закрытого ключа используется сверхвзрастащая последовательность (лёгкий способ укладки)  
Для расшифрования сообщения помимо тайного ключа (сверхвозрастающая последовательность) нужно найти обратное число A^-1 по модулю N

## Асимметричный шифр Эль-Гамаля

защищён от вероятностных атак, поскольку отправитель помимо открытого ключа получателя использует дополнительный параметр, который не нужен при расшифровании, а поэтому не известен никому, кроме отправителя  
**В алгоритме Эль-Гамаля блоку (числу) открытого сообщения соответствуют два блока (2 числа шифртекста)**  
Нужно использоваться случайные k для предоствращения вероятностных атак

## ЭЦП на основе хеш-функций

В этом случае к документу присоединяется сама ЭЦП (в отличие от прошлого алгоритма ЭЦП). В таком случае ЭЦП - зашифрованный тайным ключом отправителя хеш сообщения  
Хеш-функция - математическая функция, которая на вход принимает строку произвольной длины и фомирует строку неизменной (для данного алгоритма) длины  (как правило, 128/160/224/.../512)  
Свойства (из методы)  
Семейства MD (message digest), SHA (secure hash algorithm)  
Атака на основе парадокса дней рождения - наиболее распространённая после брутфорса на нахожденния количества попыток, которые нужно выполнить, чтобы с нужной вероятностью найти нужный хеш

Генерация ЭЦП (делает отправитель)  
На входе сообщение M, алгоритм хеширования H и тайный ключ отправителя K  
На выходе - M с подсоединённой подписью H  

1. M --H--> h(M)
2. ЭЦП = Cкто(h(M))
3. M||ЭЦП --> подписанное сообщение
4. можно ещё зашифровать публичным ключом получателя

Подтверждение подлинности  
На входе M, ЭЦП, H, публичный ключ отправителя K  

1. M' --H--> h(M')
2. дешифрация зашифрованного сообщения при помощи K h(M)
3. сравнение h(M) и h(M')
4. если они равны, сообщение действительно подписал отправитель

## Эллиптические кривые
- Уравнение: y^2 = (x^3 + ax + b) % p, p - порядок поля  
- Основные операции над ЭК связаны со **сложением** двух точек, находящихся на этой кривой  
- ЭК также принадлежит точка О, бесконечно удалённая от кривой  
- Задача, которую решает криптоаналитик, формулируется как "даны точки P, Q на ЭК порядка N (N - число точек на прямой). Нужно найти число x такое, что P = x * Q. x является дискретным логарифмом Q по основанию P"  
- Стандарт ЭЦП РБ строится на основе алгоритма Шнорра и эллиптических кривых. ЭК используются для генерации ключевой информации  

## Защита ПО в информационных системах

- ПО состоит из ИС, ...  
- Модели разграничения доступа субъектов к объектам  
  - мандатная
    - объекту назначается метка, субъекту - право (?)
  - дискреционная
    - на основе списков
    - на основе таблиц
      - все объекты ко всем субъектам
    - объекты (файлы, например)
    - субъекты
- Протокол Kerberos
  - предназначен для организации канала связи между клиентом и сервером на основе шифрования сеансовым ключом
  - реализация - клиент-сервер
  - для шифрования используется DES
  - процедура
    - клиент запрашивает у kerberos разрешение на обращение к ресурсу к службе организации таких сообщений, эта служба называется Ticket Granting
    - после анализа предоставленных документов о возможности организации сообщения между клиентом и сервером керберос выдаёт клиенту соответствующие разрешения
    - пользуясь разрешением службы керберос, клиент запрашивает TGS о выделении ему мандата на организацию канала между клиентом и сервером
    - TGS возвращает мандат
    - клиент пересылает сообщение серверу
- Протокол IPSec
  - базовый протокол обеспечения безопасности на уровне IP-соединения
  - дополняет IPv4 и является составной частью IPv6
  - возможности: 
  - основная задача - создание между двумя компьютерами, связанными через небезопасную IP сеть, безопасного туннеля
  - работает на сетевом уровне ОСИ
  - туннель создаётся с применением криптографических методов защиты информации
  - SA (Инфомрационная структура, описывающая параметры туннеля безопасности, Security Association) включает в себя
    - IP получателя
    - указание на протокоды безопасности, используемые при передаче данных
    - ключи шифрования
  - для работы нужен SSL-сертификат
- SSL и TLS обеспечивают защищённую передчу данных в компьютерной сети
  - используются в веб-браузерах, электронной почте, сервисах мгновенных сообщений и т.д.
  - сертификат - в своём роде уникальная ЭЦП сайта
    - состоит из двух частей:
      - публичная для шифрования трафика от клиента к серверу
      - приватная для расшифрования полученного от клиента зашифрованного трафика на сервере
    - на основе публичного ключа формируется запрос на создание сертификата в Центр сертификации
  - X 509 - международный документ, в котором описаны все необходимые процедуры для реализации технологий, связанных с SSL-сертификатом, в нём описана Public Key Infrastructure (PKI)

## Вирусы

- классификация
  - вирусы - саморазмножающиеся программы, дописывающие свои коды к исполняемым файлам. Могут содержать или не содержать деструктивные функции
    - макровирусы - файловые вирусы, создаются на основе Visual Basic Application, заражают офисные документы, инфицирует файл, когда он попадает в RAM. Большинство обладает свойствами резидентов и действует только во время работы с файлом
  - черви - сетевые вирусы, самостоятельно распространяющиеся по сети, и, в отличие от вирусов, не дописывающие себя к файлам. Вместо этого они нагоняют трафик, съедают ресурсы компьютера и производят утечки данных
  - кейлоггеры - программы, регистрирующие нажатия клавиш, делающие снимки рабочего стола и т.д.
  - трояны - собирают информацию с компьютера жертвы (пароли, базы данных) и тайно отправляют их злоумышленнику по сети. Есть разновидности, устанавливающие другое ПО на ПК жертвы
  - снифферы - анализаторы сетевого трафика
  - руткиты - не являются зловредами, но маскируют деструктивные программы
  - эксплойты - программы, которые через ошибки в коде ПО предоставляют несанкционированный доступ
  - звонилка - может изменять настройки уже существующих соединений
  - AdWare и SpyWare - не успел рассказать
- 5 генераций вирусов
  - простые
  - самоузнаваемые
  - скрывающиеся
  - вооружённые
  - изменяющие свойства
- методы обнаружения
  - по сигнатуре кода вируса, характерного только ему (устарел)
  - при помощи дизассемблирования
- базовые техники обнаружения
  - сканирование (машина поиска + база данных сигнатур известных вирусов)
  - мониторинг
  - контроль целостности (использует хеширование, алгоритмы шифрования DES, RSA; если контрольная сумма меняется, значит, что-то случилось)
- функциональные блоки современного вируса
  - механизм распространения
  - механизм инфекции
  - механизм репликации
  - механизм деструкции
  - дполнительные механизмы (заметание следов, изменение свойств)
